A Skeleton WebApp

daffy.io is a skeleton webapp which can be forked and used as the basis of your new site.

It uses the following technologies:

  • GoLang
  • Goth
  • BoltDB
  • GetMDL

Fork the repo, rename all the 'daffy' stuff to your own name, then you procede. See the repo on GitHub for more details: https://github.com/appsattic/daffy.io.

Go and create apps in each of the social login providers you need, configure the appropriate ENV VARS and then restart the server. You should be able to log in straight away.

  • DAFFY_BASE_URL=http://example.com
  • DAFFY_PORT=8080
  • DAFFY_TWITTER_CONSUMER_KEY=...
  • DAFFY_TWITTER_CONSUMER_SECRET=...
  • DAFFY_GPLUS_CLIENT_ID=...
  • DAFFY_GPLUS_CLIENT_SECRET=...
  • DAFFY_GITHUB_CLIENT_ID=...
  • DAFFY_GITHUB_CLIENT_SECRET=...
  • DAFFY_SESSION_AUTH_KEY_V2=...
  • DAFFY_SESSION_ENC_KEY_V2=...
  • DAFFY_SESSION_AUTH_KEY_V1=...
  • DAFFY_SESSION_ENC_KEY_V1=...

Once you are up and running, just start filling in the application itself. e.g. for an image upload service, you just need the ability for users to upload images! :) Easy aye?
